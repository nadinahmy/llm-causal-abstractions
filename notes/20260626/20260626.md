The **low-level model** is the fully-detailed description: one variable per pixel, so 10,000 variables in total (`X₁₁` through `X₁₀₀,₁₀₀`), each representing its own pixel's color.

The **high-level model** is a summary. 
- how many black pixels are in the **upper half** of the grid (_upper count_), and
- how many black pixels are in the **left half** of the grid (_left count_).
- Each of these can range from 0 to 5,000, since each half contains 5,000 pixels. These are represented by the two high-level variables `U_H` and `L_H`.

So abstraction is: throw away all 10,000 individual pixels and keep this pair. tau takes a full grid picture and returns _(upper count, left count)_.

The upper count and the left count both include the purple region. That quadrant has 2,500 pixels ---> the two counts can never be independent.

What is the problem then? Suppose I want to translate the intervention "set the upper count to _m_" while leaving the left count alone. To fix the upper count, I have to fix the pixels in the upper half. But the upper half _already includes the shared upper-left quadrant_ — and those same pixels also feed into the left count. So by fixing the top, I partly fixed the left count too, while the lower-left quadrant is still free to vary ---> ==the left quadrant isn't pinned to a single value==, and it also can't reach all of its values — it's stuck in a window of width 2,500. That's neither "left count = some specific number" nor "left count = anything," so it doesn't correspond to any "clean" high-level intervention.

What can I intervene on then? Only interventions that fix the upper half **and** the left half together ---> everything except the lower-right quadrant. Doing that pins down _both_ counts/quadrants at once, giving a clear high-level intervention of the form "upper count = _m_ **and** left count = _m′_."

That's exactly why this example is a τ-abstraction but **not** a _strong_ one. A strong abstraction requires that _every_ high-level intervention be reachable from the low level model — like for example saying "set the upper count alone." This single-variable intervention exists but no low-level intervention produces it. The abstraction is faithful for the interventions it _does_ support; ==it just doesn't support all of them==
---> Strong tau abstraction broken! 

Some conclusions from the paper:
1. Generalizing the abstraction relation to a notion of ==approximate abstraction==, given that in most real-life settings the mappings between different levels are only approximately correct.
2. Importance for explainable AI : abstracting causal models with large complexities into simpler causal models with only a few variables is directly relevant to the demand for explainable AI, usually the problem isn't the fact that no causal model is available, but in the fact that the only available model is too complicated for humans to understand.
