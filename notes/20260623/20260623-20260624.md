### ==Abstracting causal models== - My notes
- RW+ stated : They suggest that if there is an exact transformation τ from causal model M1 to M2, then we should think of M2 as an abstraction of M1, so that M2 is the high-level model and M1 is the low-level model.
- Abstractions ---> ignoring inessential differences
	- This leads to the question "what are inessential differences?"
- The value of X may depend on the values of only a few other variables.
	- It is also stated in the paper that "Formally, the equation FX maps R(U ∪ V − {X}) to R(X),so F_X determines the value of X, given the values of ==all the other variables in U ∪ V==."
	- I'm a bit confused by this as most of the research I've seen so far says that an endogenous variable is dependent on its parents and noise, not on all the other variables in a causal model. This formal definition has me intrigued, and I want to find a real example of such a scenario as I'm not able to visualize it.
- Y depends on X in ==context== u if there is some setting of the endogenous variables other than X and Y such that if the exogenous variables have value u, then varying the value of X in that context results in a variation in the value of Y ; that is, there is a setting z of the endogenous variables other than X and Y and values x and x of X such that FY (x,z,u) != FY(x',z,u).
- **The Partial Order (**≺u​**):** "chain of command" or a "solving order." If X≺u​Y, it means X comes "before" Y in the causal chain for that ==specific context==
- **Determining Values:** No loops (no A causes B while B causes A) ---> start with the variables that have no parents, find their values, and then use those to solve for the next variables in the order of the partial order until the whole system is determined
- A model is **strongly recursive** if the causal order is **independent of the context**
- Setting the value of some variables X to x in a causal model M = (S,F) results in a new causal model, denoted MX←x, which is identical to M, except that F is replaced by FX←x:for each variable Y /∈ X, F X←x Y =FY (i.e., the equation for Y is ==unchanged==), while for each X in X, the equation FX for is replaced by X= x (where x is the value in x corresponding to x).
- This part seems like important syntax to know:
	- [Y1 ← y1, . ..,Yk ← yk]ϕ says that ϕ would hold if Yi were set to yi, for i = 1,...,k.
	- (M,u) |= ψ if the causal formula ψ is true in causal model M given context u
		- A formula like [Y←y]ϕ is true if, when you replace the original structural equation for Y with the constant Y=y, the **unique solution** to the new system of equations results in ϕ being true
	- (M,u) |= X = x if the variable X has value x in the unique solution to the equations in M in context u (i.e., the unique vector of values that simultaneously satisfies all equations in M with the variables in U set to u)
- Recursion here means that information flows through the causal model in a clear one-way flow
- When I manually set a variable ([Y←y]ϕ), the model:
	1. Replaces the original equation for Y with the constant y.
	2. Calculates the **new unique solution** for the rest of the variables in the model under context u.
	3. Checks if my target condition (ϕ) is true in that new solution
Noting this explanation down as it took me some time to wrap my head around it
**1. The Interventional Truth Condition:** (M,u)⊨[Y←y]ϕ
This formula defines how I can tell if a causal statement is true.
- "[If we were to set Y to y], then ϕ would happen."
- To check if this is true in the original model M, I should first look at a modified version of that model, denoted as MY←y​. [a model where the intervention on Y has been applied]
- In this new sub-model,  the context u is fixed and all other structural equations are kept the same, but the original equation for Y is deleted and replaced with the constant Y=y.
- If the unique solution to this _new_ set of equations results in the condition ϕ being true, then the original causal formula is true in M.
- Breaking it down like this made it a lot easier for me to understand^
- M(u) : is the **entire state of the world** (the values of all internal variables V in the model) when the system is "at rest" in context u, without any outside interference.
- M(u,Y←y) is just a shorthand for the state of the world **after an intervention** is applied on Y.

- The "Lossless" Simplification**
	Lemma 2.4 states that **for any complex probabilistic causal model, there is an equivalent model that satisfies the uev assumption**.
	- A complex model where noise variables are shared and "entangled" can be transformed mathematically it into a cleaner model where every variable has its own unique noise, and because the new model is **equivalent** (∼), it will still produce the exact same causal predictions as the complex one.
### Some important syntax
- Pr({u ∈ R(U) : (M,u) |= ϕ}) = Pr'({u'∈ R(U') : (M',u') |= ϕ}) : two models are "the same" if they always predict the same probabilities for any causal event, even if their internal "low-level" details or noise sources differ
	- The Context Set: {u∈R(U):(M,u)⊨ϕ : This represents a collection of **"possible worlds"** or contexts within the first model (M).
		- u∈R(U): Refers to a specific setting of the **exogenous variables**
		- (M,u)⊨ϕ: States that in model M, given the specific context u, the causal formula ϕ is **true**.
		- The whole set just captures every possible external scenario where the rules of the model M lead to the outcome ϕ being true.
	- Pr(...) just calculates the **total likelihood** that one of the "successful" contexts will occur.
- If the above formula is satisified we can guarantee zero loss of causal information
- (M,u)⊨V=v : In model M, given the specific external conditions u [specific values of exogenous variables], the internal variables V will end up having the values v
- (M,u)⊨[Y←y] (V=v) : In model M and context u, if we were to set variable / set of variables Y to value/values y, then the collection of all variables V would end up being exactly v.
- Pr(v)=Pr({u:M(u)=v}) : to find the probability of an internal event (v), you simply sum up the probabilities of **every external context (**u**) that makes that event happen**
- PrX←x(v) = Pr({u : M(u, X ← x) =v}) : What is the chance that outcome v happens if I manually force X to be x?
	- defines how a manual intervention X←x creates a new probability distribution over the system's states. It works by aggregating the probabilities of all external contexts u that, under the modified structural equations, uniquely result in the state v. This is the standard way to ==measure the "causal effect" of an action in a probabilistic framework.==

**Micro-Model (**ML​**)**

**Macro-Model (**MH​**)**
- **Inputs:** Average variable Xˉ=n1​∑Xj​.

Problem with RW+ approach:
- **"Exact Transformation"** definition is often too weak to be a reliable measure of abstraction.
- Relying on a single probability distribution allows a high-level model to "cheat" by only being accurate in the specific scenarios that actually happen, while ignoring whether its underlying logic is actually faithful to the low-level model.
- "Exact Transformation" rule only requires that the high-level model (MH​) and low-level model (ML​) match their predictions for a **specific distribution** (PrL​)
- We end up with a high-level model that is considered an "abstraction" simply because the environment is so limited that the model's flaws are never exposed
- **Example 3.4** : Because each model's probability distribution is concentrated on a single, ==fixed scenario==, they never "see" each other's actual causal logic, they match only because they are being tested on a tiny, non-representative slice of reality
- If you only allow one or two interventions, the high-level model only has to "pass two tests" to be considered an abstraction.
- If only a few external scenarios (u) have a positive probability, MH​ only has to be correct in those specific cases.
- We're left with high-level models that are mathematically "exact" but intuitively **meaningless**, since they don't capture the actual causal structure of the system—they just happen to match the data in ==one specific setup.==
###### **"Faithful" Mapping**
###### The goal of causal abstraction simply is to see if a complex low-level model (ML​, like an LLM's internal activations) can be abstracted into a simpler high-level model (MH​, like a logic gate or a human concept).,
###### For this abstraction to be "faithful," it must pass the **Commutative Diagram test**:
###### - **Path A:** take an action at the low level, see the result, and then "zoom out" (using τ) to see the high-level outcome.
###### - **Path B:** "zoom out" the action first (using ω), perform it directly in the simple high-level model, and see the result.
###### - **The Rule:** If Path A and Path B lead to the **same answer**, the macro-model is a faithful representation of the micro-reality.

> [!IMPORTANT]
> Definition of Pr_L. Pr_L is the probability distribution on R(U_L). It defines the likelihood of the external contexts (the "noise") in the micro-model. Requiring a transformation to hold **for all** Pr_L ensures that the high-level abstraction is a property of the **system's structure**, not just an accidental match based on a specific dataset.

Example 3.10 (Super important, explains why relying solely on uniform transformations still does not guarantee correct abstraction)
- Shows how two models with **completely different causal structures** can be considered "abstractions" of each other if we are allowed to manipulate the intervention mapping (ω) arbitrarily.
- The Two Models: Different Causal Logic
	- The authors set up two simple models, M1​ and M2​, both using two binary variables (X1​,X2​)
	- M1 (causal dependency) ​: In this model, X1​ is determined by external noise (U1​), and X2​ **is caused by** X1​ (X2​=X1​). (causal dependency between X2 and X1)
	- M2​ **(Independence):** In this model, both X1​ and X2​ are independent. Each is determined by its own unique noise variable (X1​=U1​ and X2​=U2​). There is **no causal link** between them.
- These 2 models are considered transformations for each other because the we **restrict the allowed interventions** so that we never "see" the difference in their structures:
	- In M1​, you are only allowed to intervene on X1​. Because X2​ follows X1​, any change to X1​ automatically changes X2​ to match.
	- In M2​, you are only allowed to intervene on **both variables at once** to the same value. This forces them to always be identical, mimicking the behavior of M1​.
- If we look at the math behind uniform transformations, then we can say that these models are abstractions of each other ---> this contradicts the basic concept of what an "abstraction" should be:
	- The state mapping (τ) between these models is the **identity mapping** (meaning we aren't zooming out at all; X1​ is still X1​).
	- If the variables are exactly the same, we expect the interventions to be exactly the same too. But the mapping ω is "in conflict" with τ—it forces a single intervention in one model to map to a double intervention in the other just to make the math work.
	- If two models are abstractions of each other, neither is truly "higher-level." An abstraction should ==move from a complex micro-description to a simplified macro-description==, not just swap between two equally detailed but different models.
	- if the variables are the same (τ is identity), then the actions [interventions] performed on them should also be the same (ω should be identity). The fact that the math allows ω to be something other than the identity (like mapping a single intervention to a double intervention) suggests that a true "Abstraction" needs stricter rules [giving rise to τ-abstraction]
Important concept :
- identity mapping is a function where the output is exactly the same as the input (f(x)=x)
- the mapping does not change, group, or simplify the information as it moves from the low-level model to the high-level model.

##### ==**Strong** τ**-Abstraction** (Definition 3.14)==
**The Requirement:** IHτ​=IH∗​ **(Full Coverage)**
- requires that the set of high-level interventions induced by the mapping (IHτ​) must be equal to **the set of all possible high-level interventions** (IH∗​).
- The micro-model (ML​) must be detailed enough to "explain" or "ground" **every single action** you can take in the macro-model (MH​).
- If there is a "macro-action" you can imagine that cannot be mapped back to a specific set of "micro-actions" then the abstraction is **not strong**.
